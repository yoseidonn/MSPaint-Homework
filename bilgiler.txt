PROJEYİ DERLEME:
   - Terminal/Command Prompt'u proje klasöründe açın
   - Şu komutu çalıştırın: dotnet build
   - Derleme başarılı olursa "Build succeeded" mesajı görünecektir

UYGULAMAYI ÇALIŞTIRMA:
   - Yöntem 1 (Terminal'den):
     * Komut: dotnet run
     * Veya: dotnet run --project MSPaint.csproj
   
   - Yöntem 2 (Visual Studio'dan):
     * Visual Studio 2022'de projeyi açın
     * F5 tuşuna basın veya "Start" butonuna tıklayın
   
   - Yöntem 3 (Executable'dan):
     * bin\Debug\net8.0-windows\win-x64\MSPaint.exe dosyasını çalıştırın
     * Not: Self-contained build olduğu için .NET runtime dahildir

İLK ÇALIŞTIRMA:
   - Uygulama açıldığında otomatik olarak "Canvas Setup Dialog" açılacaktır
   - Canvas boyutlarını ve pixel size'ı ayarlayın
   - "OK" butonuna tıklayarak canvas'ı oluşturun

ÖZELLİKLER:
   - Pencil Tool: Serbest çizim yapmak için
   - Rectangle/Ellipse Tools: Şekil çizmek için
   - Text Tool: Metin eklemek için
   - Fill Tool: Alan doldurmak için
   - Eraser Tool: Silmek için
   - Undo/Redo: Ctrl+Z / Ctrl+Y
   - Save/Load: PNG ve BMP formatları desteklenir

EK NOTLAR
--------------------------------------------------------------------------------

1. PROJE YAPISI:
   - MVVM (Model-View-ViewModel) mimarisi kullanılmıştır
   - Model: src/Core/ klasörü (PixelGrid, CanvasSettings, ColorPalette)
   - ViewModel: src/ViewModels/ klasörü (MainViewModel, CanvasViewModel)
   - View: src/Views/ ve src/Canvas/ klasörleri (XAML dosyaları)

2. TASARIM KALIPLARI:
   - Command Pattern: Undo/Redo için (src/Commands/)
   - Strategy Pattern: File format handling için (FileService.cs)
   - Template Method Pattern: Tool sistemi için (ToolBase.cs)
   - Observer Pattern: Property change notifications (INotifyPropertyChanged)
   - RelayCommand Pattern: WPF Command binding için

3. ÖNEMLİ BİLGİLER:
   - Uygulama Windows Forms ColorDialog kullanır (renk seçimi için)
   - Unsafe code kullanılır (bitmap rendering için)
   - Self-contained build: .NET runtime dahil, bağımsız çalışır
   - Maksimum fill limiti: 100,000 pixel (performans için)

4. BİLİNEN SINIRLAMALAR:
   - Sadece Windows platformunda çalışır (WPF gereksinimi)
   - Çok büyük canvas'larda (1000x1000+) performans düşebilir
   - Fill tool maksimum 100,000 pixel doldurur

5. HATA AYIKLAMA:
   - Eğer derleme hatası alırsanız, .NET 8.0 SDK'nın yüklü olduğundan emin olun
   - "Object reference not set" hatası alırsanız, canvas'ı önce oluşturun
   - Uygulama çökerse, exception mesajı gösterilecektir

6. GELİŞTİRME NOTLARI:
   - Proje .NET 8.0 ile geliştirilmiştir
   - WPF ve Windows Forms birlikte kullanılmıştır
   - Dirty region tracking ile optimize rendering yapılmıştır
   - Command Pattern ile memory-efficient undo/redo implementasyonu
